You are an expert Python developer. Build a Binance Spot trading bot called "TradeCore Pro" with strict risk management, historical data integration, and an adaptive learning system that improves from losing trades.

**1. EXCHANGE & SETUP:**
- Use `ccxt` for Binance Spot (Testnet default, controlled by `.env` flag `TESTNET=true`).
- API keys stored in `.env`. Use `python-dotenv`.

**2. TRADING PARAMETERS (Fixed for 500 Trades/Day):**
- **Position Size**: Fixed at **$10 USDT** per trade (not % based). 
- **Concurrency**: Maximum **5 open positions** simultaneously.
- **Target Volume**: ~500 trades per day (scanning 30 pairs every 15 seconds easily hits this).
- **Stop Loss (Dynamic)**: `Entry_Price - (ATR(14) * 1.5)`.
- **Take Profit (Dynamic)**: `Entry_Price + (ATR(14) * 2.5)`.
- **Max SL Cap**: If `(ATR * 1.5) / Entry_Price > 5%`, **SKIP** the trade entirely (prevents $10 positions from losing more than $0.50 in a single trade).
- **Daily Loss Limit**: Hard circuit breaker at **-$10 USD** per day. If hit, bot halts until midnight UTC.

**3. ENTRY STRATEGY (Confidence Scoring):**
Scan every 15 seconds. Trigger a BUY only if Confidence >= 65 AND these filters pass:
- **Trend (EMA)**: 1-min EMA5 > EMA20.
- **Volume**: Current volume > 20-period average volume.
- **Pullback**: Current 1-min candle `wick_length > body_length` (buys dips).
- **Macro Context**: 1-Hour Price > 1-Hour EMA(50) (ensures bullish macro).

**4. ADAPTIVE LEARNING ENGINE (Learn from Mistakes):**
The bot must track its performance in a JSON file (`memory.json`) and use it to filter out bad trades in REAL-TIME.

- **A) Loser Pair Blacklist**: Track the last 30 trades for each symbol. If a symbol's win-rate drops below **40%** over its last 10 closed trades, add it to a blacklist and **do not trade it for 24 hours**.
- **B) Toxic Hour Filter**: Track PnL for every UTC hour (e.g., 14:00–15:00). If any hour has generated a **negative cumulative PnL** over the last 3 days, the bot must **skip entries during that hour** entirely.
- **C) Persistent Memory**: Save all trade results (Symbol, Entry, Exit, PnL, Timestamp) to `memory.json` immediately when a trade closes. Load this file on startup so the bot "remembers" losing streaks even after a restart.

**5. HISTORICAL DATA USAGE (Pre-Fetch & Backtest):**
- **Warm-Up**: On startup, fetch 200x 1-min and 100x 1-hour candles for ALL pairs via REST API. Pre-calculate indicators so the bot works instantly.
- **Backtest Mode**: If `.env` flag `BACKTEST_MODE=true`, skip live trading. Load `historical_data.csv`, run the exact strategy logic, and print: Total Trades, Win Rate (%), Total PnL ($), and Max Drawdown (%).

**6. EXECUTION (OCO Orders):**
- Place a MARKET BUY order for exactly $10 worth of the asset.
- Immediately place `STOP_LOSS` and `TAKE_PROFIT` as `reduceOnly` orders on Binance. **DO NOT use Python loops to monitor prices**—let the exchange handle exits.

**7. LOGGING & UI:**
- Print a live "Scanner" table every 15s (Symbol, Confidence, RSI, ATR%, Status).
- Print a "Trade Log" for every entry/exit with PnL.
- Print a "Daily Stats" summary every hour (Trades taken, Win-rate, Current PnL).

**8. FILES TO CREATE:**
- `main.py` (Full bot logic).
- `requirements.txt` (ccxt, pandas, numpy, python-dotenv, websockets).
- `.env.example` (Template for keys, flags, and BACKTEST_MODE).
- `memory.json` (Auto-created on first run to store trade history).

**FINAL INSTRUCTION:**
Write bulletproof, heavily commented code. Prioritize the risk management rules above all else. The bot must survive a 10-trade losing streak without blowing the account.
